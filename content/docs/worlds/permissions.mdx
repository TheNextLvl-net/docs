# Command Permissions

This page lists all current Worlds permissions.

## Permission Groups

| Permission              | Description                                                      |
|-------------------------|------------------------------------------------------------------|
| `worlds.admin`          | Grants access to all current world command permissions.          |
| `worlds.commands.save`  | Grants access to all save commands.                              |
| `worlds.commands.backup` | Grants access to all backup commands.                            |

## Linked Minecraft Permissions

Some Worlds permissions grant or include Minecraft command permissions:

| Worlds permission           | Minecraft permission                |
|-----------------------------|-------------------------------------|
| `worlds.command.save-all`   | `minecraft.command.save-all`        |
| `worlds.command.save-off`   | `minecraft.command.save-off`        |
| `worlds.command.save-on`    | `minecraft.command.save-on`         |
| `worlds.command.setspawn`   | `minecraft.command.setworldspawn`   |

For `/setworldspawn`, the command checks `minecraft.command.setworldspawn`.
Server owners can grant `worlds.command.setspawn` if their permission system expands children, or grant `minecraft.command.setworldspawn` directly.

## World Entry Permissions

World entry permissions allow you to control which players can enter specific worlds.
The permission format is:

```
worlds.enter.<world-key>
```

Example:

```
worlds.enter.example:survival
```
