# Create a New World

**Usage:**

```
/world create <key>
/world create <key> generator <generator> [<options>]
/world create <key> type <type> [<options>]
/world create <key> type flat <preset> [<options>]
/world create <key> type single-biome <biome> [<options>]
```

**Permission:** `worlds.command.create`

<Callout type="info">
  See the [Permissions page](/docs/worlds/permissions) for all command permissions.
</Callout>

You can create new worlds with the `/world create <key>` command.
The key is the required first argument and identifies the world in commands, and the server registry.
If you do not specify a generator or type, the default world settings are used.

<Callout type="info" title="Frequently asked questions">
  [What does \<key\> stand for, in command arguments?](/docs/worlds/faq#key-argument)   
  [What are keys used for?](/docs/worlds/faq#why-keys)
</Callout>

### Example

Creating a new world with the default settings:

```
/world create <aqua>example:survival</aqua>
```

## Using a flat world preset

To generate a new flat world using an existing preset, use the command  
`/world create <key> type flat <preset> ...`

World presets are a way to create worlds with specific settings.  
Here you can find a list of [available presets](/docs/worlds/presets#available-presets).

Want to learn how to create your own presets?  
Check out the [Creating a world preset](/docs/worlds/presets) documentation.

### Example

Void world example:

```
/world create <aqua>example:void</aqua> type <yellow>flat</yellow> <green>the-void</green>
```

<Callout type="tip">
  You can use the `the-void` preset instead of relying on a void generator plugin.  
  Learn more about [why you should stop using void generator plugins](/blog/void-generator-plugins).
</Callout>

## Using a world generator plugin

To generate a new world using a custom generator, use the command  
`/world create <key> generator <generator> ...`  
The generator argument takes the name of the plugin that provides the custom chunk generator or biome provider.

### Example

PlotSquared generation example:

```
/world create <aqua>city:plots</aqua> generator <yellow>PlotSquared</yellow>
```

## Defining the world type

To generate a new world using a predefined world type, use the command  
`/world create <key> type <type> ...`

World types denote the type of world you want to create.  
The following types are available:

- `normal` (previously `minecraft:noise`) - [The default world generation.](https://minecraft.wiki/w/Dimension_definition#noise)
- `flat` (previously `minecraft:flat`) - [A superflat world.](https://minecraft.wiki/w/Dimension_definition#flat)
- `large-biomes` (previously `minecraft:large_biomes`) - [All biomes are larger.](https://minecraft.wiki/w/Large_biomes)
- `amplified` (previously `minecraft:amplified`) - [Generates amplified terrain.](https://minecraft.wiki/w/Amplified)
- `single-biome` (previously `minecraft:fixed`) - [Uses one specified biome everywhere.](https://minecraft.wiki/w/Dimension_definition#fixed)
- `debug` (previously `minecraft:debug`) - [Debug all block states.](https://minecraft.wiki/w/Dimension_definition#debug)

`flat` requires a preset argument, for example `type flat the-void`.  
`single-biome` requires a biome key argument, for example `type single-biome minecraft:plains`.

### Example

Single-biome world example:

```
/world create <aqua>single_biome:deep_dark</aqua> type single-biome <yellow>minecraft:deep_dark</yellow>
```

## Additional options

All of the above commands also support the following additional options:

- `bonus-chest` - Controls whether a bonus chest is generated in the world. Defaults to `false`.
- `dimension` - Specifies the [dimension type](#available-dimensions). Defaults to `minecraft:overworld`.
- `hardcore` - If set to `true`, the world will be created in hardcore mode. Defaults to `false`.
- `seed` - Sets the world seed. If not provided, a random seed will be used.
- `structures` - Controls whether structures generate in the world. Defaults to `true`.

Options are specified as `option value` pairs, and can be used in any given order.

### Examples

Creating a normal world with structures disabled, a bonus chest, and a specific seed:

```
/world create <aqua>example:survival</aqua> type <yellow>normal</yellow> structures <green>false</green> bonus-chest <light_purple>true</light_purple> seed <gold>123456789</gold>
```

Creating a nether world:

```
/world create <aqua>example:nether</aqua> type <yellow>normal</yellow> dimension <green>the_nether</green> seed <gold>12345</gold> structures <green>true</green>
```

Creating flat and custom generator worlds:

```
/world create <aqua>example:void</aqua> type <yellow>flat</yellow> <green>the-void</green>
/world create <aqua>example:custom</aqua> generator <yellow>MyGenerator</yellow> seed <gold>12345</gold>
```

## Available Dimensions

There are following standard dimensions types:

- `minecraft:overworld` - [The primary dimension.](https://minecraft.wiki/w/Overworld)
- `minecraft:the_nether` - [The Nether dimension.](https://minecraft.wiki/w/The_Nether)
- `minecraft:the_end` - [The End dimension.](https://minecraft.wiki/w/The_End)

Datapacks can add new dimensions as well.
