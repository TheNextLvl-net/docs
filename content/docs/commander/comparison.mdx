# Plugin Comparison

This page provides an objective comparison between **Commander** and various plugin hiding alternatives.

<Callout type="info" title="Disclaimer">
  This comparison was researched to the best of our ability.  
  If you find any inaccuracies, please [open an issue](https://github.com/TheNextLvl-net/docs/issues/new) so we can correct them.
</Callout>

<Callout type="warn" title="Commander is not a plugin hider">
  Commander is designed to **manage commands** – cleaning up tab completion, configuring permissions,
  and removing commands that don't belong on your server. It is **not** designed to hide which plugins you run.

  By understanding your server and intentionally curating which commands are visible, executable, and permitted,
  Commander happens to be far more effective at controlling command exposure than any dedicated "plugin hider" ever could.
  Dedicated plugin hiders only block surface-level commands like `/plugins` without understanding your server's actual
  command landscape, leaving countless other ways to discover your plugins.

  Commander requires thought and configuration – it is not plug-and-play.
  But that is exactly why it works.
</Callout>
