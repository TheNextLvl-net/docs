The Service Convert command is one of ServiceIO's most powerful features, allowing you to migrate data between different service providers without losing any information. This is particularly useful when switching from one economy or permission plugin to another.

## Overview

The convert command provides a safe, reliable way to:
- **Transfer Economy Data** - Migrate balances, banks, and transaction history
- **Convert Permission Data** - Move groups, permissions, and user assignments
- **Preserve Relationships** - Maintain user-group associations and inheritance
- **Validate Results** - Verify data integrity after conversion

<Callout type="info" title="Safety First">
  The convert command creates backups before any conversion and validates the process at each step. Your original data is never modified until you confirm the conversion.
</Callout>

## Command Syntax

The basic command structure is:

```bash
/service convert [source] [target] [options]
```

### Parameters

- **source** - The service provider to convert data FROM
- **target** - The service provider to convert data TO  
- **options** - Additional flags to control the conversion process

To see all available service providers for conversion:


- Their types (Economy, Permission, etc.)
- Current status and version information
- Which conversions are supported

To check if a specific service supports conversion:

```bash

```

Example:
```bash

```

## Economy Conversions

### Supported Economy Providers

Common economy plugins that support conversion:

- **Essentials** - Basic economy with per-world support
- **VaultEconomy** - Simple economy implementation
- **TNE (The New Economy)** - Advanced multi-currency economy
- **CMI Economy** - Part of the CMI plugin suite
- **EconomyLite** - Lightweight economy solution

### Basic Economy Conversion

Convert from one economy plugin to another:

```bash
/service convert Essentials TNE
```

This will:
1. Scan all player balances in Essentials
2. Create equivalent accounts in TNE
3. Transfer balance data
4. Verify the conversion succeeded
5. Create a backup of original data

### Per-World Economy Conversion

For plugins with world-specific economies:

```bash
/service convert Essentials VaultEconomy --worlds
```

Options:
- `--worlds` - Include world-specific balances
- `--world [world-name]` - Convert only a specific world
- `--merge-worlds` - Combine all world balances into global balance

### Bank Account Conversion

Convert banking data between supported providers:

```bash
/service convert Essentials TNE --banks
```

This preserves:
- Bank account names and balances
- Bank member lists and permissions
- Bank ownership information
- Account creation dates

Convert permissions from one plugin to another:

This transfers:
- All user permissions
- Group definitions and permissions
- User-group assignments

## Multi-Service Conversion

Convert multiple service types simultaneously:

```bash
# Convert both economy and permissions
/service convert --economy Essentials:TNE --permissions PEX:LuckPerms

# Convert with different options per service
/service convert \
  --economy "Essentials:TNE --banks --worlds" \
  --permissions "PEX:LuckPerms --metadata"
```

## Conversion Process

### Step-by-Step Workflow

1. **Pre-Conversion Checks**
   ```bash
   /service convert Essentials TNE --dry-run
   ```
   - Validates source and target services
   - Estimates conversion time and data volume
   - Identifies potential issues or conflicts

2. **Backup Creation**
   ```bash
   /service convert Essentials TNE --backup
   ```
   - Creates timestamped backup of source data
   - Stores backup in secure location
   - Provides backup verification

3. **Data Migration**
   ```bash
   /service convert Essentials TNE --execute
   ```
   - Transfers data in batches for performance
   - Provides real-time progress updates
   - Handles errors gracefully with retry logic

4. **Validation & Verification**
   ```bash
   /service convert --verify-last-conversion
   ```
   - Compares source and target data
   - Validates data integrity
   - Reports any discrepancies

5. **Cleanup (Optional)**
   ```bash
   /service convert --cleanup-source Essentials
   ```
   - Removes old service provider data
   - Updates plugin configuration
   - Disables source plugin (if requested)

### Monitoring Progress

During conversion, monitor progress with:

```bash
# Check current conversion status
/service convert --status

# View detailed progress
/service convert --status --verbose

# Cancel running conversion
/service convert --cancel
```

## Error Handling

### Common Issues

<Tabs items={['Permission Errors', 'Data Conflicts', 'Plugin Issues']}>
  <Tab value="Permission Errors">
    **Insufficient Permissions:**
    ```
    Error: You don't have permission to convert data
    ```
    **Solution**: Ensure you have `serviceio.convert` permission
    
    **File Access Issues:**
    ```
    Error: Cannot read source data files
    ```
    **Solution**: Check file permissions and plugin data folder access
  </Tab>
  <Tab value="Data Conflicts">
    **Duplicate Accounts:**
    ```
    Error: User 'player123' already exists in target service
    ```
    **Solution**: Use `--merge` or `--overwrite` options
    
    **Invalid Data:**
    ```
    Error: Invalid balance value: -999999.99
    ```
    **Solution**: Clean source data or use `--fix-invalid` option
  </Tab>
  <Tab value="Plugin Issues">
    **Service Not Found:**
    ```
    Error: Source service 'Essentials' not detected
    ```
    **Solution**: Verify plugin is installed and enabled
    
    **Version Incompatibility:**
    ```
    Error: Target service version not supported
    ```
    **Solution**: Update to compatible version or use `--force` flag
  </Tab>
</Tabs>

### Recovery Options

If conversion fails:

```bash
# Restore from automatic backup
/service convert --restore-last

# Restore from specific backup
/service convert --restore [backup-timestamp]

# Manual recovery with partial data
/service convert --recover --partial-ok
```

## Best Practices

### Pre-Conversion Planning

1. **Test on Development Server**
   - Never run conversions directly on production
   - Set up identical test environment
   - Verify all functionality after test conversion

2. **Create Manual Backup**
   ```bash
   # Backup plugin data folders
   tar -czf plugin-backup-$(date +%Y%m%d-%H%M%S).tar.gz \
     plugins/Essentials \
     plugins/LuckPerms \
     plugins/ServiceIO
   ```

3. **Schedule Maintenance Window**
   - Plan conversion during low-activity periods
   - Notify users of potential downtime
   - Have rollback plan ready

### During Conversion

1. **Monitor Server Resources**
   - Watch memory usage during large conversions
   - Monitor disk space for backup storage
   - Check network latency if using remote databases

2. **Keep Logs**
   ```bash
   # Enable detailed logging
   /service convert --log-level debug Essentials TNE
   ```

3. **Validate Incrementally**
   ```bash
   # Check progress every 1000 records
   /service convert --validate-every 1000 Essentials TNE
   ```

### Post-Conversion

1. **Verify Critical Data**
   - Check high-balance accounts manually
   - Verify administrator permissions
   - Test core plugin functionality

2. **Update Configurations**
   - Adjust plugin settings for new service
   - Update other plugins that depend on converted service
   - Remove or disable old service provider

3. **Monitor Performance**
   - Watch server performance after conversion
   - Check for any error logs
   - Verify all dependent plugins work correctly

Required permissions for convert command:

- `serviceio.convert` - Basic conversion access
- `serviceio.convert.economy` - Convert economy data
- `serviceio.convert.permissions` - Convert permission data
- `serviceio.convert.backup` - Create/restore backups
- `serviceio.convert.admin` - Administrative functions (cleanup, force, etc.)

## Examples

### Complete Migration Example

Migrating from Essentials + PermissionsEx to TNE + LuckPerms:

```bash
# Step 1: Preview the conversion
/service convert --dry-run \
  --economy Essentials:TNE \
  --permissions PermissionsEx:LuckPerms

# Step 2: Create backup and convert
/service convert --backup \
  --economy "Essentials:TNE --banks --worlds" \
  --permissions "PermissionsEx:LuckPerms --metadata --inheritance"

# Step 3: Validate results
/service convert --verify-last-conversion

# Step 4: Clean up old data (after verification)
/service convert --cleanup-source Essentials PermissionsEx
```

### Selective Data Migration

Converting only VIP players with high balances:

```bash
/service convert Essentials TNE \
  --min-balance 10000.0 \
  --groups vip,premium,elite \
  --active-only 7 \
  --backup
```
