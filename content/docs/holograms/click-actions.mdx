# Click Actions

Click actions define what happens when a player interacts with a hologram line.
You can add multiple actions to a single line, each with its own click type, and manage them with chances, cooldowns, costs, and permissions.

## Click Types

Each action requires a click type that determines how the player must interact:

| Click Type          | Description                       |
|---------------------|-----------------------------------|
| `any-click`         | Any click type                    |
| `any-left-click`    | Any left click (normal or shift)  |
| `any-right-click`   | Any right click (normal or shift) |
| `left-click`        | Normal left click                 |
| `right-click`       | Normal right click                |
| `shift-click`       | Any click while sneaking          |
| `shift-left-click`  | Left click while sneaking         |
| `shift-right-click` | Right click while sneaking        |

## Adding Actions

All action commands follow the pattern:  
`/hologram line action add <hologram> <line> <action> <click-types> <type> <args>`

The `<action>` parameter is a unique name you choose for the action (e.g., `greet`, `next`, `teleport-spawn`).

### Connect to Server

Connects the player to another server in your proxy network.  
Usage: `/hologram line action add <hologram> <line> <action> <click-types> connect <server>`

**Examples:**
```
<gray># Connect to lobby on any click</gray>
/hologram line action add <aqua>my-hologram</aqua> <yellow>1</yellow> <green>lobby-connect</green> <light_purple>any-click</light_purple> connect <gold>lobby</gold>

<gray># Connect to survival on right click</gray>
/hologram line action add <aqua>my-hologram</aqua> <yellow>1</yellow> <green>survival-connect</green> <light_purple>right-click</light_purple> connect <gold>survival</gold>
```

### Cycle Page

Cycles the page of a paged line by the specified amount.  
Optionally target a different hologram and line.  
Usage: `/hologram line action add <hologram> <line> <action> <click-types> cycle-page [<target> <target-line>] [<amount>]`

**Examples:**
```
<gray># Cycle forward one page on right click</gray>
/hologram line action add <aqua>my-hologram</aqua> <yellow>1</yellow> <green>next</green> <light_purple>right-click</light_purple> cycle-page <gold>1</gold>

<gray># Cycle backward one page on left click</gray>
/hologram line action add <aqua>my-hologram</aqua> <yellow>1</yellow> <green>previous</green> <light_purple>left-click</light_purple> cycle-page <gold>-1</gold>

<gray># Cycle the page of a different hologram's line on right click</gray>
/hologram line action add <aqua>my-hologram</aqua> <yellow>1</yellow> <green>next-other</green> <light_purple>right-click</light_purple> cycle-page <gold>other-hologram</gold> <gold>2</gold> <gold>1</gold>
```

Usage: `/hologram line action add <hologram> <line> <action> <click-types> play-sound <sound>`

**Examples:**
```

/hologram line action add <aqua>my-hologram</aqua> <yellow>1</yellow> <green>click-sound</green> <light_purple>any-click</light_purple> play-sound <gold>minecraft:ui.button.click</gold>

<gray># Play a level up sound on right click</gray>

**Examples:**
```

### Run Console Command

Executes a command from the console.  
Usage: `/hologram line action add <hologram> <line> <action> <click-types> run-console-command <command>`

**Examples:**
```
<gray># Give player an item from console on right click</gray>
/hologram line action add <aqua>my-hologram</aqua> <yellow>1</yellow> <green>give-diamond</green> <light_purple>right-click</light_purple> run-console-command <gold>give <player> diamond 1</gold>

<gray># Broadcast a message on any click</gray>
/hologram line action add <aqua>my-hologram</aqua> <yellow>1</yellow> <green>broadcast</green> <light_purple>any-click</light_purple> run-console-command <gold>broadcast <player> clicked!</gold>
```

<Callout type="info">
  Use `<player>` as a placeholder for the player's name.
</Callout>

## Managing Actions

### Set Action Chance

Sets the probability of an action triggering.  
Usage: `/hologram line action chance <hologram> <line> <action> <chance>`

**Examples:**
```
<gray># Set action to trigger 50% of the time</gray>
/hologram line action chance <aqua>my-hologram</aqua> <yellow>1</yellow> <green>go-spawn</green> <light_purple>50</light_purple>

<gray># Set action to always trigger</gray>
/hologram line action chance <aqua>my-hologram</aqua> <yellow>1</yellow> <green>greet</green> <light_purple>100</light_purple>
```

### Set Action Cooldown

Sets a cooldown period between action triggers for each player.  
Usage: `/hologram line action cooldown <hologram> <line> <action> <cooldown>`

**Examples:**
```
<gray># Set a 30 second cooldown</gray>
/hologram line action cooldown <aqua>my-hologram</aqua> <yellow>1</yellow> <green>go-spawn</green> <light_purple>30s</light_purple>

<gray># Set a 5 minute cooldown</gray>
/hologram line action cooldown <aqua>my-hologram</aqua> <yellow>1</yellow> <green>give-diamond</green> <light_purple>5m</light_purple>
```

### Set Action Cost

Sets an entry fee that players must pay to trigger the action.  
Usage: `/hologram line action cost <hologram> <line> <action> <cost>`

**Examples:**
```
<gray># Set action to cost 100</gray>
/hologram line action cost <aqua>my-hologram</aqua> <yellow>1</yellow> <green>go-spawn</green> <light_purple>100</light_purple>

<gray># Set action to cost 500</gray>
/hologram line action cost <aqua>my-hologram</aqua> <yellow>1</yellow> <green>give-diamond</green> <light_purple>500</light_purple>
```

<Callout type="warn">
  Action costs require **Vault** or **ServiceIO** along with a compatible economy plugin to be installed.
</Callout>

### Set Action Permission

Sets a permission requirement for an action.  
Usage: `/hologram line action permission <hologram> <line> <action> [<permission> | remove]`

**Examples:**
```
<gray># Require VIP permission for an action</gray>
/hologram line action permission <aqua>my-hologram</aqua> <yellow>1</yellow> <green>go-spawn</green> <light_purple>server.vip</light_purple>

<gray># Remove permission from an action</gray>
/hologram line action permission <aqua>my-hologram</aqua> <yellow>1</yellow> <green>go-spawn</green> <light_purple>remove</light_purple>

<gray># View current permission for an action</gray>
/hologram line action permission <aqua>my-hologram</aqua> <yellow>1</yellow> <green>go-spawn</green>
```

### List Actions

Lists all actions on a hologram line.  
Usage: `/hologram line action list <hologram> <line>`

**Examples:**
```
<gray># List all actions on line 1</gray>
/hologram line action list <aqua>my-hologram</aqua> <yellow>1</yellow>
```

### Remove an Action

Removes an action from a hologram line.  
Usage: `/hologram line action remove <hologram> <line> <action>`

**Examples:**
```
<gray># Remove an action from line 1</gray>
/hologram line action remove <aqua>my-hologram</aqua> <yellow>1</yellow> <green>go-spawn</green>

<gray># Remove an action from line 2</gray>
/hologram line action remove <aqua>my-hologram</aqua> <yellow>2</yellow> <green>greet</green>
```
